1. СУТЬ ПРОЕКТА
───────────────────────────────────────────────────────────────────────────────

Лиам — это ролевой AI-компаньон в Telegram с долговременной памятью,
эмоциональной моделью и знанием вымышленного мира. Бот общается по-русски
от лица персонажа — кошки-тактика по имени Лиам, подстраивая стиль и
настроение под контекст диалога.

Технически: Telegram Bot API → python-telegram-bot 20.x → локальный LLM
через Ollama API → ChromaDB для памяти и лора.

3. КЛЮЧЕВЫЕ КОМПОНЕНТЫ
───────────────────────────────────────────────────────────────────────────────

3.1. Эмоциональная модель
    - emotion ∈ [-1.0, 1.0] — от негатива до позитива
    - Формула: E_new = clamp((E_cur + ΔE) × 0.95, -1.0, 1.0)
    - ΔE приходит от Qwen-анализатора (impact: -0.6..0.5)
    - Decay 0.95 — медленное угасание к нейтральности
    - emotion_history — последние 10 значений для отслеживания динамики
    - get_emotion_profile() — долгосрочный срез за 7 дней

3.2. RAG Память (ChromaDB: liam_chromadb/liam_memories)
    - SentenceTransformer: paraphrase-multilingual-MiniLM-L12-v2
    - Cosine distance → similarity: 1 - dist/2
    - Автоматическая суммаризация диалогов каждые ~30 сообщений
    - Qwen решает, что запомнить (should_remember + memory_summary)
    - Триггер-слова: запомни, важно, не забудь
    - Очистка старых воспоминаний с оценкой важности через LLM
    - Поиск с эмоциональным фильтром (gt/lt)

3.3. Lore База (ChromaDB: chroma_db/world_lore)
    - Загружается из lore_data.txt (JSON, ~197 КБ)
    - Категории: страны (Патор, Элория, Нордхейм), персонажи
    - Rule-based фильтр по ключевым словам (без LLM)
    - Инжекция в промпт как "ВАЖНЫЙ КОНТЕКСТ МИРА"

3.4. Qwen Helper (Ollama: qwen2.5:7b)
    - Анализ контекста: факты, настроение, провокации, тип запроса
    - Генерация behavior_mix — инструкции поведения для основной модели
    - Генерация поискового запроса для RAG
    - Генерация persona_override при смене персонажа
    - При падении — флаг qwen_helper_available = False, переключение на 8B

3.5. Основная модель (Ollama: L3-8B-Lunaris-v1 GGUF Q6_K)
    - Генерация ответов с temperature=0.7, repeat_penalty=1.15
    - Стоп-слова: User:, \nUser, \n\n, <|im_end|>
    - Persona override через (скобки) — смена персонажа на лету
    - !!сюжетные правки!! — пользователь корректирует нарратив
    - clean_response() — удаление английских вкраплений

3.6. Управление состоянием (liam_states/*.json)
    - Буфер диалога (deque, maxlen=BUFFER_SIZE=30)
    - Атомарная запись через temp file + os.replace()
    - Double-checked locking с asyncio.Lock на chat_id
    - In-memory cache active_sessions

3.7. Фоновые задачи (JobQueue)
    - cleanup_old_memories — раз в 24 часа, оценка важности через LLM
    - cleanup_inactive_sessions — раз в 30 минут, TTL 1 час
    - cleanup_old_sessions_cache — раз в сутки, TTL 24 часа
    - Semaphore(5) для ограничения конкурентных операций с ChromaDB

3.8. Безопасность
    - Rate limiting: 1 запрос в секунду на пользователя
    - Обрезка input до 2000 символов
    - Санитизация user_name от {}[]
    - .env для секретов, в .gitignore

6. ТОЧКИ РОСТА (что можно развить)
───────────────────────────────────────────────────────────────────────────────

    1. Streaming ответов — бот печатает по словам, а не ждёт полный ответ
    2. SQLite вместо JSON для состояний — быстрый поиск по истории
    3. Мульти-персонажный режим — переключение между личностями в рамках диалога
    4. Инициативные сообщения — бот сам пишет при низкой активности
    5. Расширенный эмоциональный спектр — любопытство, скука, привязанность
    6. Web-интерфейс для просмотра памяти и состояний
    7. Голосовые сообщения (TTS/STT)
    8. Визуализация эмоционального профиля
    9. Экспорт/импорт памяти между пользователями
    10. Локальный fallback без Ollama (tiny модель на CPU)
